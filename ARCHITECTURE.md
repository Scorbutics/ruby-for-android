# Architecture - CRuby Cross-Platform

## Overview

Cross-compile CRuby 3.1.1 (MRI) as a Position Independent Executable (PIE) for Unix-based
platforms including Android 8+ (API 26) and iOS.

Forked from https://github.com/sfrieske/ruby-for-android5 (2015, now outdated).

## Build System

CMake-based with Docker containerization. Follows standard Linux convention:

```
./configure [options] → make → make install
     ↓                    ↓           ↓
Generate Makefile    CMake build   Export artifacts to ./target/
```

### Dependency Chain

```
ncurses → readline → ruby
         openssl  ↗
         gdbm    ↗
```

| Dependency | Version | Purpose |
|------------|---------|---------|
| Ruby       | 3.1.1   | Main CRuby interpreter |
| OpenSSL    | 1.1.1m  | SSL/TLS support |
| GDBM       | 1.18.1  | DBM database support |
| ncurses    | 6.4     | Terminal control |
| readline   | 8.1     | Line editing/history |

### CMake Targets per Dependency

Generated by `add_external_dependency()`:
`{lib}_download`, `{lib}_extract`, `{lib}_patch`, `{lib}_configure`,
`{lib}_build`, `{lib}_install`, `{lib}_clean`, `{lib}`

Ruby-specific: `ruby_archive` (create zip), `ruby` (build + archive)

Global: `all`, `clean`, `clean-libs`, `clean-artifacts`, `clean-downloads`, `clean-all`

### Two-Layer CMake Design

```
cmake/core/        Generic build system (reusable across projects)
  Main.cmake         declare_application()
  BuildHelpers.cmake add_external_dependency(), create_archive_target()
  PlatformDetection.cmake
  Validation.cmake
  platforms/         Android.cmake, Linux.cmake, macOS.cmake, iOS.cmake

cmake/ruby-app/    Ruby application layer
  dependencies/      Per-dependency configs (ncurses, readline, gdbm, openssl, ruby)
  patches/           Organized by {lib}/{platform}/series + *.patch
```

## Patch Categories

1. **Android SONAME versioning removal** — Android's linker expects unversioned `.so` files
2. **Ruby type redefinition fix** (Ruby 3.0.0 only, not needed for 3.1.1+)
3. **OpenSSL NDK r22+ support** (OpenSSL 1.1.1k only, not needed for 1.1.1m+)
4. **GDBM GCC 10 link fix** (Linux only)

## Build Artifacts

`./target/ruby_full-{platform}-{arch}.zip` contains:
- All shared libraries (libruby, libssl, libcrypto, libgdbm, libncurses, libreadline)
- Complete Ruby 3.1.0 stdlib + native extensions
- Ruby executables (ruby, irb, gem, rake, bundle)
- `setup_ruby.sh` environment setup script

## Runtime Deployment

```sh
# Extract, setup environment, run
. /path/to/ruby/setup_ruby.sh /path/to/ruby 3.1.0 aarch64
ruby -v
```

Environment variables set by `setup_ruby.sh`:
`LD_LIBRARY_PATH`, `PATH`, `GEM_PATH`, `GEM_HOME`, `GEM_SPEC_CACHE`, `RUBYLIB`

## Toolchain Params

Single-line CMake argument files (e.g., `arm64-v8a-android-toolchain.params`):
```
-DANDROID_STL=c++_shared -DANDROID_PLATFORM=android-26 -DANDROID_ABI=arm64-v8a
-DHOST=aarch64-linux-android -DHOST_SHORT=android-arm64
```

Custom architectures: create new `{arch}-{platform}-toolchain.params` file.

## Adding New Dependencies

1. Create `cmake/ruby-app/dependencies/{dep}.cmake`
2. Use `add_external_dependency()` function
3. Add patches in `cmake/ruby-app/patches/{dep}/{platform}/` with `series` file
4. Add dependency name to `APP_DEPENDENCIES` in `CMakeLists.txt`

## Cross-Compilation Glossary

- **Host**: Machine running the build (e.g., x86_64 Linux)
- **Target**: Machine running the binary (e.g., aarch64 Android)
- **DESTDIR**: Staging directory for install (not final runtime path)
- **PIE**: Position Independent Executable (required for Android 5+)
- **SONAME**: Shared object name embedded in ELF binary
